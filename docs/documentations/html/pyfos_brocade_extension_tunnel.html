class pyfos_brocade_extension_tunnel.extension_circuit(dictvalues=None)¶

Class of extension_circuit

Description extension_circuit

Represents circuit interface for extension blade or system.

Important class members of extension_circuit:

Attribute Name Description Frequently Used Methods
circuit-id Circuit identifier. In case of 7810 the range is restricted to 0..5. peek_circuit_id() set_circuit_id()
name The name of the interface. peek_name() set_name()
local-ha-ip-address Specifies the local HA IP address to use for the circuit. This HA configuration is only supported on 7840 and SX6. peek_local_ha_ip_address() set_local_ha_ip_address()
local-ip-address Specifies the local IP address to use for the circuit. peek_local_ip_address() set_local_ip_address()
keep-alive-timeout Specifies the keep alive timeout in milliseconds. If the tunnel does not already have FICON Emulation enabled, circuits created on the tunnel default to the the keep alive timeout of 6000 ms (6 seconds). If FICON emulation is enabled on the extension tunnel when a circuit is created, the keep alive timeout defaults to 1000 ms (1 seconds). peek_keep_alive_timeout() set_keep_alive_timeout()
maximum-communication-rate Maximum Communication Rate of the tunnel in kbps. You can set a minimum and a maximum for the committed rate to configure the tunnel for Adaptive Rate Limiting (ARL), which allows for a more effective sharing of bandwidth between applications. The maximum committed rate can be no larger than five times the minimum committed rate, and both sides of the circuit must have matching configurations. The range on 7810 is restricted to 0|20000..2500000. peek_maximum_communication_rate() set_maximum_communication_rate()
remote-ha-ip-address Specifies the remote HA IP address to use for the circuit. peek_remote_ha_ip_address() set_remote_ha_ip_address()
is-admin-enabled Is Circuit admin status enabled. Default: true, Values supported are true/false. false - Disabled true - Enabled peek_is_admin_enabled() set_is_admin_enabled()
failover-group-id Specifies the failover group ID for the configured circuit. The circuit failover groups must be defined at both ends of the extension tunnel and each failover group should include at least one metric 0 and one metric 1 circuit. If all metric 0 circuits in the failover group go down, the transmission fails over to the metric 1 circuits in the group. If all metric 0 circuits in a tunnel go down, by default the traffic will run over the metric 1 circuits. This field has no meaning when load level for the tunnel is ‘spillover’. peek_failover_group_id() set_failover_group_id()
arl-algorithm-mode Specifies Adaptive Rate Limiting (ARL) algorithm. Allowable modes are: 0 : auto 1 : reset 2 : step-down 3 : timed-step-down peek_arl_algorithm_mode() set_arl_algorithm_mode()
metric Specifies the metric for the configured circuit. A lower metric assigns a higher priority to the circuit. As data is flowing through the extension tunnel, it automatically traverses the lowest metric cost circuits. For example, if a tunnel has four circuits, three of which are set to a metric of 0 and one is set to a metric of 1, all data will flow over the metric 0 circuits. This parameter is meaningful only, if you configure more than one circuit. 0 : Primary 1 : Failover peek_metric() set_metric()
minimum-communication-rate Minimum Communication Rate of the tunnel in kbps. You can set a minimum and a maximum for the committed rate to configure the tunnel for Adaptive Rate Limiting (ARL), which allows for a more effective sharing of bandwidth between applications. The maximum committed rate can be no larger than five times the minimum committed rate, and both sides of the circuit must have matching configurations. The range on 7810 is restricted to 0|20000..2500000. peek_minimum_communication_rate() set_minimum_communication_rate()
remote-ip-address Specifies the remote IP address to use for the circuit. peek_remote_ip_address() set_remote_ip_address()
admin-enabled Is Circuit admin status enabled. Default: 1, Values supported are 0..5 However user is not allowed to set all of them as some of them are Internal RO states. Supported configure values are Disabled and Enabled only. Internal values are Config, HA-disabled, Fenced and Test. 0 - Disabled 1 - Enabled 2 - Config 3 - HA-disabled 4 - Fenced 5 - Test. Deprecated: Please use ‘is-admin-enabled’ leaf instead. peek_admin_enabled() set_admin_enabled()
dscp The Internet Protocol (IP) Differentiated Services Code Point markings for frames. peek_dscp() set_dscp()
l2-cos L2 class of service/priority as defined by IEEE 802.1p for the tunnel. peek_l2_cos() set_l2_cos()
ip-priority-high Specifies the DSCP value for IP High Priority. The range is 0 to 63. peek_dscp_ip_priority_high() set_dscp_ip_priority_high()
fc-priority-high Specifies the DSCP value for FC High Priority. The range is 0 to 63. peek_dscp_fc_priority_high() set_dscp_fc_priority_high()
ip-priority-high Specifies the L2CoS value for IP High Priority Traffic. peek_l2_cos_ip_priority_high() set_l2_cos_ip_priority_high()
fc-priority-medium Specifies the L2CoS value for FC Medium Priority Traffic. peek_l2_cos_fc_priority_medium() set_l2_cos_fc_priority_medium()
fc-priority-low Specifies the DSCP value for FC Low Priority. The range is 0 to 63. peek_dscp_fc_priority_low() set_dscp_fc_priority_low()
ip-priority-low Specifies the L2CoS value for IP Low Priority Traffic. peek_l2_cos_ip_priority_low() set_l2_cos_ip_priority_low()
fc-priority-high Specifies the L2CoS value for FC High Priority Traffic. peek_l2_cos_fc_priority_high() set_l2_cos_fc_priority_high()
fc-priority-low Specifies the L2CoS value for FC Low Priority Traffic. peek_l2_cos_fc_priority_low() set_l2_cos_fc_priority_low()
priority-control Specifies the Layer 2 Class of Service (L2CoS) value for F-Class Traffic. This priority setting controls connections between switches. peek_l2_cos_priority_control() set_l2_cos_priority_control()
ip-priority-medium Specifies the DSCP value for IP Medium Priority. The range is 0 to 63. peek_dscp_ip_priority_medium() set_dscp_ip_priority_medium()
priority-control Specifies the DSCP value for F-Class Traffic. The range is 0 to 63. peek_dscp_priority_control() set_dscp_priority_control()
ip-priority-medium Specifies the L2CoS value for IP Medium Priority Traffic. peek_l2_cos_ip_priority_medium() set_l2_cos_ip_priority_medium()
fc-priority-medium Specifies the DSCP value for FC Medium Priority. The range is 0 to 63. peek_dscp_fc_priority_medium() set_dscp_fc_priority_medium()
ip-priority-low Specifies the DSCP value for IP Low Priority. The range is 0 to 63. peek_dscp_ip_priority_low() set_dscp_ip_priority_low()
path-mtu-discovered Discovered Path MTU value. This field is valid only when Path MTU discovery is enabled. peek_path_mtu_discovered()
vlan-id Specifies the VLAN ID. When not set, this value will show up as 0. peek_vlan_id()
admin-status Is Circuit admin status enabled. Default: 1, Values supported are 0..5 However user is not allowed to set all of them as some of them are Internal RO states. Supported configure values are Disabled and Enabled only. Internal values are Config, HA-disabled, Fenced and Test. 0 - Disabled 1 - Enabled 2 - Config 3 - HA-disabled 4 - Fenced 5 - Test. peek_admin_status()
circuit-status Circuit operational status peek_circuit_status()

name The name of the interface. peek_name() set_name()
ficon-tape-write-timer Defines a time limit for pipelined write chains. This value is specified in milliseconds (ms). If a pipelined write chain takes longer than this value to complete, the ending status for the next write chain will be withheld from the channel. This limits processing to what the network and device can support. Too small a value limits pipelining performance. Too large a value results in too much data being accepted for one device on a path. This field is valid only if FICON emulation is enabled on the specified tunnel. peek_ficon_tape_write_timer() set_ficon_tape_write_timer()
