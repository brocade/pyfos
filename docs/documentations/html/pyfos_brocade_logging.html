pyfos_brocade_logging - PyFOS module to provide REST support for configuring the logging facilities in the FC switch.¶

The pyfos_brocade_logging module provides REST support for the FC switch logging facilities.

class pyfos_brocade_logging.audit(dictvalues={})¶
This class provides methods or options to retrieve and configure
audit logging.

Important Class Members:

Attribute Name Desc. Frequently Used Methods
audit-enabled Audit flag. peek_audit_enabled() set_audit_enabled()
severity-level
Severity
level.
 peek_severity_level() set_severity_level()
filter-class-list /filter-class Class list. peek_filter_class_list_filter_class() set_filter_class_list_filter_class()

Object Methods

get(session)¶
Returns a :class:`audit` object or a list of objects with attributes gathered from auditcfg.

Each object can be printed using pyfos_util.response_print(), and individual attributes can be accessed through peek methods.

patch(session)¶

Apply configurable attribute(s) within the object to audit.

Below Is an Example Using Individual Sets:

# initialize the audit object
audit_obj = pyfos_brocade_logging.audit()
# set audit enabled to true
audit_obj.set_audit_enabled(true)
# set the severity level to info
audit_obj.set_severity_level(info)
audit_obj.patch(session)

:param session: The session handler returned by the audit
:func:`pyfos_auth.login`.
:rtype: A dictionary of errors or success information.

Attribute Methods

set_audit_enabled(enable/disable)¶

Sets the audit log enabled flag to true or false (disabled) in the object.

Parameters:enable/disable – True to enable and false to disable audit logs.
Return type:A dictionary of errors or success response.
peek_audit_enabled()¶

Reads the audit log enabled state in the object.

Return type:True or false.
set_severity_level(level)¶

Sets the severity level in the object.

Parameters:level – The valid severity level (info, warning, critical, error).
Return type:A dictionary of errors or a success response.
peek_severity_level()¶

Reads the severity level in the object.

Return type:The severity level of the audit log.
set_filter_class_list_filter_class(filter_class_list)¶

Sets the filter class list in the object.

Parameters:level – The valid filter class list (security, configuration, zone, firmware, fabric, ls, cli, maps).
Return type:A dictionary of errors or a success response.
peek_filter_class_list_filter_class()¶

Reads the filter class list in the object.

Return type:The filter class list of audit logs.

patch(session)¶

Applies configurable attribute(s) within the object to log quiet control.

Below Is an Example Using Individual Sets:

# initialize the log_quiet_control object
obj = pyfos_brocade_logging.log_quiet_control()
# set the log-type attribute to enable the
# wwn based pid on the log_quiet_control
obj.set_keep_alive_period(true)
# execute HTTP patch command to apply the object to the
# log_quiet_control connected through session
obj.patch(session)

Below Is an Example of Combining Object Initialization and Attribute Sets:

# initialize the log_quiet_control object and
# set the enable attribute
obj = pyfos_brocade_logging.log_quiet_control(
    {"quiet-enabled" : true})
# execute HTTP patch command to apply the object to the
# log_quiet_control connected through session
obj.patch(session)
Parameters:session – The session handler returned by pyfos_auth.login().
Return type:A dictionary of errors or success information.

class pyfos_brocade_logging.syslog_server(dictvalues={})¶
This class provides methods and options to retrieve, configure, modify,
and delete the syslog.

Important Class Members:

Attribute Name Description Frequently Used Methods
server The syslog server name. peek_server() set_server()
port The port syslog server. peek_port() set_port()
secure-mode The secure mode using the TLS protocol. peek_secure_mode() set_secure_mode()

Object Methods

get(session)¶
Returns a :class:`syslog_server` object or list of objects with syslog server attributes.

Each object can be printed using pyfos_util.response_print(). and individual attributes can be accessed through peek methods.

post(session)¶

Adds members to a list attribute of the object. This method is used to add syslog servers.

Example Usage of the Method to Configure a New Syslog Server:

# initialize the syslog server object syslog_server_obj = pyfos_brocade_logging.syslog_server() # set the IP address of server syslog_server_obj.set_server(“10.10.10.10”)

Parameters:session – The session handler returned by pyfos_auth.login().
Return type:A dictionary of errors or a success response.
patch(session)¶

Applies configurable attribute(s) within the object to the syslog server.

Below Is an Example Using Individual Sets:

# initialize the syslog server object
syslog_server_obj = pyfos_brocade_logging.syslog_server()
# set the IP address of server
syslog_server_obj.set_server("10.10.10.10")
# set the port number
syslog_server_obj.set_port("1234")
# set secure mode to true
syslog_server_obj.set_secure_mode(true)
syslog_server_obj.patch(session)

:param session: The session handler returned by
    :func:`pyfos_auth.login`.
:rtype: A dictionary of errors or a success response.
delete(session)¶

Deletes members from a list attribute of the object. This method deletes the configured syslog server.

Example Usage of the Method to Create New Mappings:
# initialize the syslog server object
syslog_server_obj = pyfos_brocade_logging.syslog_server()
# set the IP address of server
syslog_server_obj.set_server("10.10.10.10")
# delete the syslog server
syslog_server_obj.delete(session)
