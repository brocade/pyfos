zoning_cfg_create_add - PyFOS util for specific Zoning use case.¶

The zoning_cfg_create_add provides for a specific Zoning use case.

This module is a stand-alone script and API that can be used to create new cfg(s) or add to existing cfg(s).

  • Inputs:
    • -L=<login>: Login ID. If not provided, an interactive

      prompt will request one.

    • -P=<password>: Password. If not provided, an interactive

      prompt will request one.

    • -i=<IP address>: IP address.

    • –name=<cfg name>: string name of new cfg.

    • –members=<member list>: list of members separated by “;”.

      Multiple members need to be enclosed by “”.

    • -f=<VFID>: VFID or -1 if VF is disabled. If unspecified,

      a VFID of 128 is assumed.

  • Outputs:
    • Python dictionary content with RESTCONF response data.
zoning_cfg_create_add.cfgcreate(session, cfgs)¶

Create new cfg(s) with member(s).

Example usage of the method:

cfgs = [
            {
                "cfg-name": name,
                "member-zone": {"zone-name": members}
            }
          ]
result = cfgcreate(session, cfgs)
Parameters:
  • session – session returned by login
  • cfgs – an array of cfg and members
Return type:

dictionary of return status matching rest response

Use cases

  1. Create new CFG(s) with provided member(s).
  2. Add new member(s) to existing CFG(s).
