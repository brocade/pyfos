Object Methods

get(session, name=None)¶

Returns a fibrechannel_diagnostics object or a list of objects filled with attributes gathered from the switch. If an optional name is given, either an object matching the name of the port is returned or an empty object is returned if no match is found.

Each object can be printed using pyfos_util.response_print(), and individual attributes can be accessed through peek methods.

Parameters:session – session handler returned by pyfos_auth.login()
Return type:a fibrechannel object if a name is given or a list of objects if there are more than one
patch(session)¶

Applies configurable attribute(s) within the object to a port of the switch.

Below is an example using individual sets:

# once d_port is enabled on the port
# set the payload
diag_info = pyfos_switchfcport.fibrechannel_diagnostics()
diag_info.set_name(name)
diag_info.set_payload_pattern_payload("305402420")
result = diag_info.patch(session)
pyfos_util.response_print(result)

# on enabled port, restart the test
diag_info = pyfos_switchfcport.fibrechannel_diagnostics()
diag_info.set_name(name)
diag_info.set_diagnostic_control(pyfos_switchfcport.DIAG_RESTART)
result = diag_info.patch(session)
pyfos_util.response_print(result)

Below is an example of combining object initialization and attribute sets:

# once d_port is enabled on the port
# set the payload
diag_info = pyfos_switchfcport.fibrechannel_diagnostics(
    {"name" : name,
    "payload-pattern" : {"payload" : "305402420"}})
result = diag_info.patch(session)
pyfos_util.response_print(result)

# on enabled port, restart the test
diag_info = pyfos_switchfcport.fibrechannel_diagnostics(
    {"name" : name,
    "diagnostic-control" : pyfos_switchfcport.DIAG_RESTART})
result = diag_info.patch(session)
pyfos_util.response_print(result)
