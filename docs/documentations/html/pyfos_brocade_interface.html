class pyfos_brocade_interface.extension_ip_interface(dictvalues=None)¶

Class of extension_ip_interface

Description extension_ip_interface

Represents the IP interface defined on extension blade or system.

Important class members of extension_ip_interface:

Attribute Name Description Frequently Used Methods
name The name of the interface. peek_name() set_name()
dp-id Extension Data Path Processor ID associated with the IP interface. Based on platform either it will have a single DP or dual DP. In case of single DP only DP0 is supported, and in case of dual DP both DP0 and DP1 are supported 0 : DP0 1 : DP1. peek_dp_id() set_dp_id()
ip-address Specifies the source IPv4/IPv6 address of the interface. peek_ip_address() set_ip_address()
ip-prefix-length The prefix length operator for the IP address. Once set, prefix length cannot be changed. peek_ip_prefix_length() set_ip_prefix_length()
vlan-id Specifies the VLAN ID. This operand is optional. peek_vlan_id() set_vlan_id()
mtu-size Specifies the maximum transmission unit size. The permitted range is 1280 Bytes to 9216 Bytes or the special value 1 which is equivalent to ‘auto’ mode to enable PMTU discovery. The discovered value of MTU in ‘auto’ mode can be found as a circuit attribute. peek_mtu_size() set_mtu_size()
status-flags IP interface Flags: U = Up B = Broadcast D = Debug L = Loopback P = Point2Point R = Running I = InUse N = NoArp PR= Promiscous M = Multicast S = StaticArp LU= LinkUp peek_status_flags()

Object methods

get(session, name=None)¶

Returns a fibrechannel object or a list of objects filled with attributes gathered from switch. If optional name is given, either an object matching the name of the port is returned or an empty object is returned if no match is found.

Each object can be printed using pyfos_util.response_print() and individual attributes accessed through peek methods.

Parameters:session – session handler returned by pyfos_auth.login()
Return type:a fibrechannel object if name is given or a list of objects if there are more than one
patch(session)¶

Apply configurable attribute(s) within the object to a port of the switch.

Below is an example using individual sets:

# initialize the switch object
switch = pyfos_switch.fibrechannel()
# set the enabled-state attribute to
# enable the port 0/0 on the switch
switch.set_name("0/0")
switch.set_enabled_state(pyfos_switchfcport.enableState)
# execute HTTP patch command to apply the object to the
# switch connected through session
switch.patch(session)

Below is an example of combining object initialization and attribute sets:

# initialize the switch object and set the
# enable-state attribute
switch = pyfos_switch.fibrechannel({"name" : "0/0",
    "enabled-state" : pyfos_switch.enableState})
# execute HTTP patch command to apply the object to the
# switch connected through session
switch.patch(session)
Parameters:session – session handler returned by pyfos_auth.login()
Return type:a dictionary of error or success information

When GE port protocol is configured for LAN, the GE port is connected on the LAN side for Extending LAN IP traffic over Extension tunnels using IP QOS priorities. When the GE port protocol is configured for WAN then the GE ports are connected on the WAN side and are used for configuring the FCIP/Extension tunnels using them. In case of chassis system if the slot is powered off or in faulty state then the leaf will be skipped in display. Supported values: LAN : GE port is configured for LAN side connections. WAN : GE port is configured for WAN side connections. peek_protocol() set_protocol()

portchannel-member-timeout The dynamic portchannel member timeout of the gigabit-ethernet interface. default long. peek_portchannel_member_timeout() set_portchannel_member_timeout()
auto-negotiation-enabled Auto-negotiation is enabled by default in 1G mode. In 10G mode it is disabled and not supported. When the port is set for 1G mode, you can disable auto-negotiation. In case of chassis system if the slot is powered off or in faulty state then the leaf will be skipped in display. Supported values: true : Auto negotiation mode for GE port is enabled. false : Auto negotiation mode for GE port is disabled. peek_auto_negotiation_enabled() set_auto_negotiation_enabled()
lldp-enabled-state LLDP state of the port Possible values are: true - LLDP is enabled on the port false - LLDP is disabled on the port peek_lldp_enabled_state() set_lldp_enabled_state()
lldp-profile LLDP profile name configured on the port. Blank lldp-profile means LLDP profile is not configured on this port. In such case, lldp global parameters are in use for this port. To configure a new profile or change existing profile on the port, user should perform a PATCH operation with the profile name. class pyfos_brocade_interface.portchannel(dictvalues=None)¶

Class of portchannel

Description portchannel

The list of portchannel interfaces on the switch, their related configuration and operational state.

Important class members of portchannel:

Attribute Name Description Frequently Used Methods
speed The speed of the portchannel interface. The speed of the portchannel is the speed of its member ports. speed is an read-only parameter of a port-channel. Note: The speed of all members of a portchannel must match. peek_speed()
portchannel-type The type of a portchannel interface. Portchannel Type is a mandatory parameter while the portchannel is created. Possible values: static : Static portchannel type. dynamic : Dynamic portchannel type. set_portchannel_type() peek_portchannel_type()
gigabit-ethernet-member-ports/port The list of gigabit ethernet interfaces that are members of the portchannel. set_gigabit_ethernet_member_ports_port() peek_gigabit_ethernet_member_ports_port()
primary-member-gigabit-ethernet-port The primary member interface of the portchannel. peek_primary_member_gigabit_ethernet_port()
name The portchannel name. set_name() peek_name()
operational-status-enabled The portchannel operational status. The possible values are: false : portchannel is offline. true : portchannel is online. peek_operational_status_enabled()
key The portchannel key. The portchannel key cannot be modified once the portchannel is created. If user does not provide the key during portchannel creation, the system will auto-assign an available key. The portchannel key is unique across all portchannel interfaces on the switch. set_key() peek_key()
auto-negotiation-enabled Indicates the auto-negotiation of the FCIP portchannel. The possible values are: false : auto-negotiation is off. true : auto-negotiation is on. set_auto_negotiation_enabled() peek_auto_negotiation_enabled()
lacp-system-mac-address The LACP MAC address of the system. peek_lacp_system_mac_address()
admin-state-enabled Indicates the admin state of the portchannel. The possible values are: false : portchannel is disabled. true : portchannel is enabled. set_admin_state_enabled() peek_admin_state_enabled()
