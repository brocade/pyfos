pyfos_rule_manager - PyFOS module to provide rule based handling Support.¶

The pyfos_rule_manager Provides the rule management for REST supported classes.

Introduction¶

The module pyfos_rule_manager supports rules a method for config management support. User can write rules based on various classes/object and attribute with conditionals for config operations.

Table for all predefined rules¶

Rule Pyfos Class Details
R1 defined_configuration Save the Zoning configuration after POST Rule-R1
R2 defined_configuration Save the Zoning configuration after DELETE Rule-R2
R3 defined_configuration Save the Zoning configuration after PATCH Rule-R3
R4 extension_ip_route Ignore non static route entries for DELETE Rule-R4
R5 extension_ip_route Ignore non static route entries for POST Rule-R5
R6 fibrechannel Disable/Enable ports while applying configs Rule-R6
R7 effective_configuration Handle the effective configuration for PATCH Rule-R7
R8 fibrechannel_switch Disable the switch after applying the configs Rule-R8
R9 n_port_map Delete all n-port-map instance configured fports Rule-R9
R10 n_port_map Patch all n-port-map instance configured fports Rule-R10
R11 port_group Delete all port-groups except for pg0 Rule-R11
R12 port_group Create all port-groups other than pg0 Rule-R12
R13 port_group Patch only port-group instance for pg0 Rule-R13

default(obj)¶

Implement this method in a subclass such that it returns a serializable object for o, or calls the base implementation (to raise a TypeError).

For example, to support arbitrary iterators, you could implement default like this:

def default(self, o):
    try:
        iterable = iter(o)
    except TypeError:
        pass
    else:
        return list(iterable)
    # Let the base class default method raise the TypeError
    return JSONEncoder.default(self, o)
